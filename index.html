Vorwort

Dieses Skript wurde für die Vorlesung "Programmierung und Design von WebApplications mit HTML5, CSS3 und JavaScript" geschrieben. Es stellt die Weiterentwicklung des bisherigen Skripts dar das 2011 von Florian Rappl erstellt wurde. Das alte Skript war nicht auf mobilen Geräten benutzbar und auch nicht gut als Nachschlagwerk geeignet, weswegen es das erklärte Ziel dieses Skripts ist genau dies möglich zu machen.

Die Vorlesung setzt nur geringfügige Kenntnisse in HTML voraus. Für CSS sind keinerlei Grundkenntnisse notwendig, allerdings kann vorhandenes Wissen sehr hilfreich sein. JavaScript wird von Grund auf eingeführt - hier sind nur Programmierkenntnisse in einer beliebigen (Skript-) Sprache notwendig, damit die Probleme in der Sprache und nicht in der Programmierung an sich auftreten.

Das Skript wird gerade von Grund auf neu geschrieben, wer Fehler findet soll darauf aufmerksam machen. Auch konstruktive oder unkonstruktive Kritik was Aufbau, Geschwindigkeit, Schwerpunkte und sonstige Qualitätsmerkmale angeht ist willkommen.

Die Übungsaufgaben sind über den Link oben auf der Seite zu finden. Die Lösungen zu den Übungsaufgaben sind dort ebenfalls zu finden.

Ich möchte vor allem neue, interresante Möglichkeiten von HTML5 und Javascript aufzeigen, viele davon werden noch keine breite Browserunterstützung genießen. Wenn man experimentelle Features in der Lehre meidet führt das nur dazu, dass der Kurs sehr schnell veraltet ist. Die Web Entwicklung bewegt sich sehr schnell; es kann passieren, dass unmögliche Dinge in 6 Monaten machbar sind und in zwei Jahren Standard. Grundsätzlich werde ich deswegen in Google Chrome arbeiten, da dieser Browser den anderen meist ein gutes Stück vorraus ist was die Implementierung und Optimierung von neuen Features angeht.

Neben den HTML5 Fähigkeiten von Desktop-Browsern sind die Möglichkeiten mit den mobilen Browsern im Fokus der Web-Entwickler. Im Internet gibt es jede Menge Webseiten mit Browservergleichen und Demonstrationen von aktuellen Möglichkeiten mit dem neuen Standard. Ein auf jeden Fall aufzurufender Test ist auf der Seite von HTML5Test zu finden. Sollte der verwendete Browser hier zu wenig Punkte holen, wäre ein Browserupgrade für die Vorlesung auf jeden Fall zu empfehlen.

Louis Ritzkowski, Februar 2019.

Um was gehts hier genau?

Wir werden uns mit drei Technologien beschäftigen:

  • HTML(5) zur Beschreibung der Seite (Struktur)
  • JavaScript(ES6) für die eigentlichen Interaktionen (Programmierung)
  • CSS(3) zur Gestaltung dieser (Design)

Wie bereits erwähnt werden diese Technologien schrittweise eingeführt. Daneben werden wir uns bei Zeit noch mit folgenden Möglichkeiten beschäftigen:

  • Spiele und Animationen / Grafiken (auch 3D)
  • mit JavaScript Abwärtskompatibilität zu älteren Webbrowsern schaffen
  • Webseiten durch einfache Tricks optimieren (weniger Requests, JavaScript minifizieren, ...)
  • JavaScript für die Client- und Serverentwicklung mit Node.js

Code Examples

Im Skript werden interaktive Codebeispiele in folgenden Formaten verwendet:

Javascript

                    function(input, output) {
                        console.log('bar');
                        output.value += input.value;
                    }

Der Aufbau des Javascript Editors besteht aus drei Textareas die mit etwas JS und CSS erweitert sind.

  • Input/Output: einfache Textareas ohne Erweiterung.
  • CodeEditor: Textarea deren Inhalt als Javascript Code interpretiert wird. Sie erhält input und output als variablen übergeben und kann diese modifizieren. Unter verwendung des 'this' Keywords kann sie sich auch selbst editieren.
  • Run Code: Der Knopf zur linken des Code Editors führt den interpretierten Inhalt aus. Wenn mal nichts passiert -> Console überprüfen, wahrscheinlich hat ein Semantikfehler das Ausführen verhindert. (Syntaxfehler sollte der Code Editor selbst schon anzeigen)

HTML

Der Html Editor ist grundlegend gleich aufgebaut.

  • CSS: Textarea deren Inhalt als CSS interpretiert wird. <body> kann HIER nicht gestyled werden!
  • CodeEditor: Textarea deren Inhalt als HTML interpretiert wird.
  • Zurücksetzem: Setzt HTML und CSS Eingaben zurück!
